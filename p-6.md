# Práctica. Interfaz de escritorio en Python para monitoreo y control de un ESP32 mediante Socket.IO

## Objetivo

Desarrollar una aplicación de escritorio en **Python** utilizando **Tkinter** y la biblioteca **Socket.IO**, capaz de conectarse a un servidor en **Node.js**, recibir en tiempo real el valor de una fotoresistencia enviado desde un **ESP32** y enviar comandos al servidor para controlar el encendido y apagado del LED interno y de un LED externo del microcontrolador. La interfaz deberá integrarse al ecosistema ya existente, conviviendo con la página web desarrollada previamente.

---

## Introducción

En prácticas anteriores se desarrolló un sistema distribuido en el que un **ESP32** se conecta a un **servidor Node.js**, envía datos provenientes de una fotoresistencia y recibe comandos para controlar salidas digitales. Esos datos y comandos fueron visualizados y manipulados desde una **interfaz web**, lo que permitió comprender la relación entre dispositivo, servidor e interfaz de usuario.

Posteriormente, se trabajó con distintos elementos de interfaz gráfica en **Tkinter**, explorando widgets como etiquetas, cajas de texto, botones, listas desplegables, cajas de verificación, menús, temporizadores, barras de progreso y otros componentes fundamentales para construir aplicaciones de escritorio en Python. 

En esta práctica se dará el siguiente paso: se agregará un **cliente de escritorio en Python**, el cual formará parte del mismo ecosistema. Esto significa que ya no existirá únicamente una interfaz web, sino que el mismo servidor podrá atender simultáneamente a varios tipos de cliente:

* el **ESP32**, que envía datos y ejecuta acciones;
* el **servidor Node.js**, que coordina la comunicación;
* la **interfaz web**, que ya forma parte del sistema;
* y ahora un **cliente de escritorio en Python**, que también podrá monitorear y controlar.

Con ello se busca que el alumno comprenda que una arquitectura cliente-servidor no depende de un solo tipo de interfaz. Un mismo sistema puede tener varias interfaces conviviendo al mismo tiempo, siempre que compartan el mismo protocolo de comunicación y respeten los mismos identificadores de eventos.

## Descripción general del sistema

La lógica general del sistema será la siguiente:

1. El **ESP32** lee el valor de la fotoresistencia.
2. El **ESP32** envía ese valor al **servidor Node.js**.
3. El **servidor** reenvía ese dato a todos los clientes conectados.
4. Tanto la **interfaz web** como el **cliente de escritorio en Python** pueden mostrar el mismo valor en tiempo real.
5. Desde el **cliente Python**, el usuario podrá enviar comandos al servidor.
6. El **servidor** reenviará esos comandos al **ESP32** para encender o apagar LEDs.

Con esto, el sistema quedará conformado por múltiples nodos que colaboran dentro del mismo ecosistema.

## Bloque 6. Conexión al servidor Socket.IO

### Propósito

Construir la lógica que permitirá conectar y desconectar el cliente Python del servidor.

### Código

```python
def conectar_socket():
    global socket_cliente, conectado, hilo_escucha

    if conectado:
        messagebox.showinfo("Información", "El cliente ya está conectado.")
        return

    ip = entry_ip.get().strip()
    puerto_texto = entry_puerto.get().strip()

    if not ip:
        messagebox.showwarning("Advertencia", "Escriba la IP del servidor.")
        return

    if not puerto_texto:
        messagebox.showwarning("Advertencia", "Escriba el puerto del servidor.")
        return

    try:
        puerto = int(puerto_texto)
    except:
        messagebox.showerror("Error", "El puerto debe ser numérico.")
        return

    def tarea_conexion():
        global socket_cliente, conectado

        try:
            root.after(0, lambda: agregar_log(f"Intentando conectar a {ip}:{puerto} ..."))

            socket_cliente = SocketIO(ip, puerto)

            # ====================================================
            # AQUÍ SE REGISTRAN LOS IDS DE EVENTOS SOCKET.IO
            # Deben coincidir con el servidor Node.js
            # ====================================================
            socket_cliente.on(EVENTO_FOTORESISTENCIA, recibir_fotoresistencia)
            socket_cliente.on(EVENTO_ESTADO_LEDS, recibir_estado_leds)

            conectado = True
            root.after(0, lambda: actualizar_estado_conexion("Conectado", "green"))
            root.after(0, lambda: agregar_log("Conexión establecida correctamente."))

            while conectado:
                socket_cliente.wait(seconds=1)

        except Exception as e:
            conectado = False
            root.after(0, lambda: actualizar_estado_conexion("Desconectado", "red"))
            root.after(0, lambda: agregar_log(f"Error de conexión: {e}"))
            root.after(0, lambda: messagebox.showerror("Error", f"No fue posible conectar.\n{e}"))

    hilo_escucha = threading.Thread(target=tarea_conexion, daemon=True)
    hilo_escucha.start()


def desconectar_socket():
    global socket_cliente, conectado

    if not conectado:
        messagebox.showinfo("Información", "El cliente no está conectado.")
        return

    try:
        conectado = False
        if socket_cliente is not None:
            socket_cliente.disconnect()
        actualizar_estado_conexion("Desconectado", "red")
        agregar_log("Conexión cerrada.")
    except Exception as e:
        agregar_log(f"Error al desconectar: {e}")
```

### Explicación

En este bloque ocurre lo siguiente:

* se leen la IP y el puerto escritos por el usuario;
* se valida que el puerto sea numérico;
* se crea la conexión con el servidor;
* se registran los eventos que el cliente escuchará;
* se mantiene un ciclo de espera para recibir mensajes;
* la conexión se ejecuta en un hilo aparte para no congelar la interfaz.

### Aspecto importante

Aquí se indica claramente dónde se registran los eventos escuchados:

```python
socket_cliente.on(EVENTO_FOTORESISTENCIA, recibir_fotoresistencia)
socket_cliente.on(EVENTO_ESTADO_LEDS, recibir_estado_leds)
```

Esos identificadores deben coincidir con el servidor.

### Resultado esperado

El alumno dispone de funciones para conectarse y desconectarse del sistema distribuido.

## Bloque 7. Envío de comandos al servidor

### Propósito

Permitir que el cliente Python envíe órdenes para controlar los LEDs del ESP32.

### Código

```python
def enviar_led_interno(estado):
    if not conectado or socket_cliente is None:
        messagebox.showwarning("Advertencia", "Primero debe conectarse al servidor.")
        return

    try:
        # ====================================================
        # ID DEL EVENTO SOCKET.IO PARA LED INTERNO
        # Debe coincidir con el servidor Node.js
        # ====================================================
        socket_cliente.emit(EVENTO_LED_INTERNO, estado)
        agregar_log(f"Enviado {EVENTO_LED_INTERNO}: {estado}")
    except Exception as e:
        agregar_log(f"Error al enviar comando LED interno: {e}")


def enviar_led_externo(estado):
    if not conectado or socket_cliente is None:
        messagebox.showwarning("Advertencia", "Primero debe conectarse al servidor.")
        return

    try:
        # ====================================================
        # ID DEL EVENTO SOCKET.IO PARA LED EXTERNO
        # Debe coincidir con el servidor Node.js
        # ====================================================
        socket_cliente.emit(EVENTO_LED_EXTERNO, estado)
        agregar_log(f"Enviado {EVENTO_LED_EXTERNO}: {estado}")
    except Exception as e:
        agregar_log(f"Error al enviar comando LED externo: {e}")


def enviar_mensaje_prueba():
    if not conectado or socket_cliente is None:
        messagebox.showwarning("Advertencia", "Primero debe conectarse al servidor.")
        return

    mensaje = entry_mensaje.get().strip()

    if not mensaje:
        messagebox.showwarning("Advertencia", "Escriba un mensaje de prueba.")
        return

    try:
        socket_cliente.emit(EVENTO_MENSAJE_PRUEBA, mensaje)
        agregar_log(f"Enviado {EVENTO_MENSAJE_PRUEBA}: {mensaje}")
    except Exception as e:
        agregar_log(f"Error al enviar mensaje de prueba: {e}")
```

### Explicación

Estas funciones emiten eventos hacia el servidor mediante `emit()`. Cada botón de la interfaz llamará a una de estas funciones.

### Dónde se envían los eventos

En estas líneas:

```python
socket_cliente.emit(EVENTO_LED_INTERNO, estado)
socket_cliente.emit(EVENTO_LED_EXTERNO, estado)
socket_cliente.emit(EVENTO_MENSAJE_PRUEBA, mensaje)
```

Aquí es donde el alumno debe revisar si los nombres coinciden con los usados en su servidor.
